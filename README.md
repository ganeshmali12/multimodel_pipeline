# Multimodal Mindspace: Mental Health Pipeline + Inference API

This project contains a full workflow for:
- synthetic multimodal mental-health dataset generation,
- dynamic feature selection pipeline,
- multi-model training and comparison,
- secure FastAPI inference service.

## Project Overview

The workflow is centered around `automated_pipeline.ipynb`, which:
1. Loads and validates dataset schema.
2. Detects label columns dynamically (`mental_health_label` or `*_label`).
3. Performs one-hot label encoding when needed.
4. Runs EDA and quality checks.
5. Performs two-stage feature selection:
   - low-variance removal (`variance < 0.05`),
   - correlation-based redundancy filtering.
6. Trains multiple models and selects the best by Accuracy/F1.
7. Exports model artifacts and reports.

## Inference API

### 1. Configure API key

Create `.env` in project root (if not already present):

```env
MODEL_API_KEY=your-strong-secret-key
```

`model_api.py` loads this key via:
- `load_dotenv()`
- `os.getenv("MODEL_API_KEY")`

### 2. Run API

```powershell
.\venv\Scripts\Activate.ps1
uvicorn model_api:app --host 127.0.0.1 --port 8000 --reload
```

### 3. API endpoints

Open endpoint:
- `GET /health`

Protected endpoints (require header `X-API-Key`):
- `GET /metadata`
- `GET /sample-payload`
- `POST /predict`
- `POST /predict-batch`

### 4. Example request

```powershell
curl.exe -H "X-API-Key: your-strong-secret-key" http://127.0.0.1:8000/metadata
```

For prediction:
- use `GET /sample-payload` to get full feature template,
- fill all feature values,
- post to `POST /predict`.

## Notes on Label Encoder File

`LABEL_ENCODER_PATH = BASE_DIR / "best_model_label_encoder.pkl"` is a compatibility fallback.

- If model predicts class names directly (current GradientBoosting workflow), this file is not required.
- If future models predict encoded numeric class IDs, label encoder can decode IDs back to class names.

## Troubleshooting

### Import warnings in editor but code runs
Ensure VS Code interpreter points to:
- `C:\multimodel_Mindspace\venv\Scripts\python.exe`

### API returns key not configured
- Check `.env` exists in project root.
- Confirm `MODEL_API_KEY` is present.
- Restart `uvicorn` after changing `.env`.

### 401 invalid/missing API key
- Send header exactly as:
  - `X-API-Key: <your key>`
